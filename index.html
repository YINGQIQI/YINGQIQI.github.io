meta标签

响应式布局—meta标签

1、汉字申明编码

该编码是汉字申明编码。 

<meta charset='utf-8' />

2、优先使用 IE 最新版本和 Chrome

<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1" />

3、浏览器内核控制

目前我国国内浏览器的内核基本都是双内核（webkit和trident）

<meta name="renderer" content="webkit|ie-comp|ie-stand">

4、页面关键字

“your tags”最好不要超过12个，页面关键字十分重要，可被搜索引擎的爬虫程序收录，用于用户搜索

<meta name="keywords" content="your tags，” />

5、页面描述

页面描述尽量不要超过120个字（包括标点符号），同样会被搜索引擎的爬虫程序收录检测，用于用户搜索

<meta name=“description"content="120｜ 150 words" />

6、搜索引擎索引方式

搜索引擎的引入方式的content值有六种：all none、index、follow、noindex、nofollow
这六个值的含义是：

all：文件将被检索，且页面上的链接可以被查询；

none：文件将不被检索，且页面上的链接不可以被查询；

index：文件将被检索；

follow：页面上的链接可以被查询；

noindex：文件将不被检索；

nofollow：页面上的链接不可以被查询。

<meta name=“robots” content="index,follow" />

7、页面重定向和刷新

content内的数字代表时间（秒），既多少时间后刷新，如果加url,则会重定向到指定网页（搜索引擎能够自动检测，也很容易被引擎视作误导而受到惩罚）

<meta http-equiv="refresh" content="0;url=" />

8、网页作者信息

<meta name="author" content="author name" />

CSS3单位

em/rem

  • em单位是相对于父级的单位，可用于设置字体的大小和容器的宽高
  • 含义：em表示的含义是倍数，例如2em表示的含义为父级像素的两倍;rem单位是相对于html文档的单位，可用于设置字体的大小，不可设置容器的宽高
  • 举例：

代码如下所示：

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>单位</title>
</head>
<body>
<div class="div1" style="font-size:12px;">
    <h1>我是h1</h1>
    <h2>
        <div class="div3" style="font-size:2em">我是div3</div>
    </h2>
    <div class="div2" style="font-size:2em;width:3em;height:4em">我是div2</div>
</div>    
<div class="div4" style="font-size:4rem" >我是div4</div>
</body>
</html>

浏览器中的效果如下图所示：

分析：

1、 div1的字号设置为12px；

2、 h1的字号本身默认为2em，则h1的字号为24px；

3、 h2的字号默认为1.5em。div3的字号为36px；

4、 div2的字号为2em，其父级为div1（字号为12px），则div2的字体为24px；

5、 div4的字号为html默认字体16px的4倍：64px；

vw、vh

  • vw和vh是相对于视觉窗口的，用于设置容器大小;

  • 含义：作为宽和高的含义，vm和vh代表的是视觉窗口的百分比，如：width：50vw表示占视觉窗口的百分之五十；当浏览器窗口缩小的时候，容器的大小也会随着变小，一般用于响应式的网页设计。

  • 举例：

代码如下所示：

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>单位</title>
</head>
<body>
<div class="div1" style="height:50vh;width:50vw;background:red;"></div>
</body>
</html>

浏览器效果如图所示，div的长和宽始终保持为浏览器的百分之五十：

px

  • px属性是最常用的单位，即可以设置字号大小也可以设置容器的宽高。

属性

maxwidth、maxheight、minheight、minwidth属性

  • maxwidth和maxheight用来设置容器等元素的最大宽和高，当元素小于最大宽和高的值时，元素保持为自身的宽高，当元素大于最大宽或高的值时，元素会适应容器的最大宽和高进行调整，不超过maxheight和maxwidth的值。
  • minheight和minwidth用来设置容器等元素的最小宽和高，当元素大于最小宽和高的值时，元素保持为自身的宽高，当元素小于最小宽或高的值是，元素会适应容器的最小宽和高进行调整，扩充到最小宽和高。
  • maxwidth、maxheight、minheight、minwidth属性的值一般为一个固定像素值。
  • 仍以上面的百分比的代码案例来分析：给每个li设置了最小宽度，当浏览器缩小到每个li小于200px时，li由于设置了最小宽度不会再继续缩小，而会自动变成两行，把最右边的li挤到下一行。

Hexo 博客搭建步骤

安装nodeJS：

  • 进入nodeJS官网下载nodeJS (官网地址：https://nodejs.org/en)
  • 安装成功后在终端进行测试：输入node -v，回车查看，如下图所示则安装成功

安装hexo：

  • 在终端输入npm install hexo -g,然后回车
  • 若此时终端出现root Administrator等错误时，需重新输入sudo npm install hero -g，以管理员方式安装，回车。
  • 输入管理员密码，回车
  • 测试hexo是否安装成功：在终端输入hexo，回车查看，如在终端出现如下图所示的代码，则安装成功。

创建目录并进入(以桌面创建为例):

  • 在终端输入“cd Desktop”,回车,进入电脑桌面
  • 在终端输入“mkdir blog”，回车，在桌面创建一个名为blog的文件夹
  • 在终端输入“cd blog”,回车，进入名为blog的文件夹
  • 在终端输入“hexo init”，回车，初始化hexo的目录
  • 在终端输入“hexo server”回车，启动本地服务，在成功启动本地服务器后会出现一个本地博客地址：http://localhost:4000/,点击进入。(按住ctrl+c退出hexo运行)，如下图所示：
  • 在进入hexo博客的网站后注册一个账号并登陆，步骤如下图所示：

1、进入注册界面，点击右上角的“sign in”

2、填写用户名、注册邮箱和密码

注册github账号：

  • 首先需要配置ssh-key的值：在终端输入“ssh-keygen”，连续回车三次,如下图所示:

  • 在终端输入“vim ~/.ssh/id_rsa.pub”复制ssh-key的值。

  • 打开本地博客地址，点击用户名旁边的settings，点击personal settings 里面的SSH and GPG keys，在key的输入框内把之前复制的ssh-key的值输入进去，点击ADD SSH key确认。这样一个博客账号就创建好了。



  • 在终端输入“：q”，回车，进入博客文件。

  • 下面将配置github与本地机器验证：
    在终端输入“git config –global user.name”用户名”（例如：git config –global user.name”YINGQIQI”）
    在终端输入“git config –global user.email”邮箱地址”（例如：git config –global user.email”2544360251@qq.com”）
  • 新建博客域名仓库：


    [注意：]repositary的名字必须和用户名一致

配置config.yml：

打开桌面上的blog文件夹中的config.yml文件 在deploy字段的下面的type后面填入“ git”；进入自己本地博客地址把域名仓库的 https的域名地址复制到deploy字段下面的repo后面

发布：

  • 在终端输入“cd Desktop”，回车,然后输入“cd blog”，回车，进入blog文件夹中
  • 在终端输入：“npm install hexo-deployer-git –save”
  • 在终端输入：“hexo g”，回车，把一些文件转换成css和html文件
  • 在终端输入：“hexo d”，回车，把生成的网页发布到仓库。
  • 打开博客地址查看效果

博客使用：

进入博客

新建博客的网址为：https://**用户名**.github.io/

更换博客主题

  • 可以在网上下载一些hexo博客的主题，clone主题的https地址
  • 在终端进入blog里面的themes文件夹中，回车，然后输入“git clone 主题http地址”回车（此过程需要一段时间），例如下图所示：
  • 此时打开blog文件夹的themes中可以发现多出一个主题文件夹
  • 在blog文件夹的_config.yml文件中修改主题并保存

    更换博客名称

  • 在_config.yml文件中可以修改主题和博客题目以及作者语言等，在修改完后需保存并重新在终端运行

更新博客

  • 在终端输入“hexo new”,创建新日志
  • 在终端输入“hexo server”，在本地服务器查看博客内容是否有问题
  • 在终端输入“hexo g”把md文件转化成html文件和css文件
  • 在终端输入“hexo d”把本地写的博客发布到仓库
