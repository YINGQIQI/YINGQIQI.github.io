html5新增标签：

概述

html5语义标签不同于html4标签，没有了行块级之分，新增的标签都默认设置了display：block属性；只是在原来的基础上使布局语义更加清晰，使机器阅读更加简单。

布局标签

  • header标签：其语义为文档头部的意思，一般用于html的头部

  • nav标签：其语义为导航栏的意思，一般用于html的导航栏，一个网页可以有多个导航栏。

  • section标签：其语义为聚合标签，一般用于文档的内容主体中，把相关的内容 包裹关联起来，一般不建议section里面嵌套section。

  • artical标签：其语义也为聚合标签，不同于section，artical一般把相关的section都聚合起来，形成文档的主体部分。在布局中artical可以嵌套artical使用。相比于原来学的div标签，section标签和artical标签语义更加规范明确。

  • aside标签：其语义为侧边栏的意思，它的内容独立于网页的内容主体，一般用作广告位等。

  • footer标签:其语义为页脚的意思，一般用作网页的底部布局

  • main标签：其语义为文章主体，但现在不经常用此标签

  • address标签：其语义为地址的意思，一般出现在html文档的footer元素的其他信息位置，用来定义文档作者/所有者的联系信息，如果 address 元素位于 元素内部，则它表示该文档作者/所有者的联系信息；如果address元素位于
    元素内部，则它表示该文章作者/所有者的联系信息

  • hgroup标签：其语义为标题组的意思，一般用作文章新闻的主标题和副标题，一般在有超过两个标题的情况下用hgroup标签。

    <!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>html5</title>
</head>
<body>
    <header></header>
    <nav></nav>
    <article>
        <section>
            <hgroup>
                <h1></h1>
                <h2></h2>
            </hgroup>
        </section>
    </article>
    <footer></footer>
</body>
</html>

MAMP的使用

MAMP的作用

  • MAMP的使用可以让我们把本地PC端的html文件在手机端打开
  • 其中在使用过程中可以搭配二维码插件，在移动端只需要扫一扫便可以轻松打开

MAMP使用步骤

  • 第一步：在本地MAMP中的htdocs文件夹中创建、书写html文件
  • 第二步：打开MAMP让它启动（start server），打开Open Webstart page里面的“my website”

  • 第三步：打开所要打开的html文件，在终端输入“ifconfig”，找到自己本地网址所对应的网络地址，把前面打开的html文件中的url的localhost转换成网络地址


  • 第四步：把最终地址在chrome浏览器中打开，并用手机扫描二维码即可 （ 注意：需要在同一个局域网中实现）

在使用MAMP的过程中，为了使内容可以适应手机屏幕可以使用一个meta标签

<meta name="viewport" content="width=device-width, initial-scale=1.0,maximum-scale=1.0, user-scalable=no"/>

CSS3单位

em/rem

  • em单位是相对于父级的单位，可用于设置字体的大小和容器的宽高
  • 含义：em表示的含义是倍数，例如2em表示的含义为父级像素的两倍;rem单位是相对于html文档的单位，可用于设置字体的大小，不可设置容器的宽高
  • 举例：

代码如下所示：

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>单位</title>
</head>
<body>
<div class="div1" style="font-size:12px;">
    <h1>我是h1</h1>
    <h2>
        <div class="div3" style="font-size:2em">我是div3</div>
    </h2>
    <div class="div2" style="font-size:2em;width:3em;height:4em">我是div2</div>
</div>    
<div class="div4" style="font-size:4rem" >我是div4</div>
</body>
</html>

浏览器中的效果如下图所示：

分析：

1、 div1的字号设置为12px；

2、 h1的字号本身默认为2em，则h1的字号为24px；

3、 h2的字号默认为1.5em。div3的字号为36px；

4、 div2的字号为2em，其父级为div1（字号为12px），则div2的字体为24px；

5、 div4的字号为html默认字体16px的4倍：64px；

vw、vh

  • vw和vh是相对于视觉窗口的，用于设置容器大小;

  • 含义：作为宽和高的含义，vm和vh代表的是视觉窗口的百分比，如：width：50vw表示占视觉窗口的百分之五十；当浏览器窗口缩小的时候，容器的大小也会随着变小，一般用于响应式的网页设计。

  • 举例：

代码如下所示：

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>单位</title>
</head>
<body>
<div class="div1" style="height:50vh;width:50vw;background:red;"></div>
</body>
</html>

  • px属性是最常用的单位，即可以设置字号大小也可以设置容器的宽高。

属性

maxwidth、maxheight、minheight、minwidth属性

  • maxwidth和maxheight用来设置容器等元素的最大宽和高，当元素小于最大宽和高的值时，元素保持为自身的宽高，当元素大于最大宽或高的值时，元素会适应容器的最大宽和高进行调整，不超过maxheight和maxwidth的值。
  • minheight和minwidth用来设置容器等元素的最小宽和高，当元素大于最小宽和高的值时，元素保持为自身的宽高，当元素小于最小宽或高的值是，元素会适应容器的最小宽和高进行调整，扩充到最小宽和高。
  • maxwidth、maxheight、minheight、minwidth属性的值一般为一个固定像素值。

meta标签

Veröffentlicht am 2016-06-24

响应式布局—meta标签

1、汉字申明编码

该编码是汉字申明编码。 

<meta charset='utf-8' />

2、优先使用 IE 最新版本和 Chrome

<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1" />

3、浏览器内核控制

目前我国国内浏览器的内核基本都是双内核（webkit和trident）

<meta name="renderer" content="webkit|ie-comp|ie-stand">

4、页面关键字

“your tags”最好不要超过12个，页面关键字十分重要，可被搜索引擎的爬虫程序收录，用于用户搜索

<meta name="keywords" content="your tags，” />

5、页面描述

页面描述尽量不要超过120个字（包括标点符号），同样会被搜索引擎的爬虫程序收录检测，用于用户搜索

<meta name=“description"content="120｜ 150 words" />

6、搜索引擎索引方式

搜索引擎的引入方式的content值有六种：all none、index、follow、noindex、nofollow
这六个值的含义是：

all：文件将被检索，且页面上的链接可以被查询；

none：文件将不被检索，且页面上的链接不可以被查询；

index：文件将被检索；

follow：页面上的链接可以被查询；

noindex：文件将不被检索；

nofollow：页面上的链接不可以被查询。

<meta name=“robots” content="index,follow" />

7、页面重定向和刷新

content内的数字代表时间（秒），既多少时间后刷新，如果加url,则会重定向到指定网页（搜索引擎能够自动检测，也很容易被引擎视作误导而受到惩罚）

<meta http-equiv="refresh" content="0;url=" />

8、网页作者信息

<meta name="author" content="author name" />
