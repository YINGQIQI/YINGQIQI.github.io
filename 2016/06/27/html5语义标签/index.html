html5语义标签

Veröffentlicht am 2016-06-27

html5新增标签：

概述

html5语义标签不同于html4标签，没有了行块级之分，新增的标签都默认设置了display：block属性；只是在原来的基础上使布局语义更加清晰，使机器阅读更加简单。

布局标签

  • header标签：其语义为文档头部的意思，一般用于html的头部

  • nav标签：其语义为导航栏的意思，一般用于html的导航栏，一个网页可以有多个导航栏。

  • section标签：其语义为聚合标签，一般用于文档的内容主体中，把相关的内容 包裹关联起来，一般不建议section里面嵌套section。

  • artical标签：其语义也为聚合标签，不同于section，artical一般把相关的section都聚合起来，形成文档的主体部分。在布局中artical可以嵌套artical使用。相比于原来学的div标签，section标签和artical标签语义更加规范明确。

  • aside标签：其语义为侧边栏的意思，它的内容独立于网页的内容主体，一般用作广告位等。

  • footer标签:其语义为页脚的意思，一般用作网页的底部布局

  • main标签：其语义为文章主体，但现在不经常用此标签

  • address标签：其语义为地址的意思，一般出现在html文档的footer元素的其他信息位置，用来定义文档作者/所有者的联系信息，如果 address 元素位于 元素内部，则它表示该文档作者/所有者的联系信息；如果address元素位于
    元素内部，则它表示该文章作者/所有者的联系信息

  • hgroup标签：其语义为标题组的意思，一般用作文章新闻的主标题和副标题，一般在有超过两个标题的情况下用hgroup标签。

    <!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>html5</title>
</head>
<body>
    <header></header>
    <nav></nav>
    <article>
        <section>
            <hgroup>
                <h1></h1>
                <h2></h2>
            </hgroup>
        </section>
    </article>
    <footer></footer>
</body>
</html>

    以上是一个简单的网页布局代码
    默认属性：

    article, aside, footer, header, hgroup, main, nav, section {
display: block;
}

  • mark标签（高亮文本，常用作于搜索高亮）

    <p>原来你是我最想留住的<mark>幸运</mark>原来我们和爱情考得那么近</p>

    默认属性：

  • meter标签（定义度量衡，常用于投票占比使用量等，仅用于已知最大和最小值的度量。）

    <meter min="0" max="100" value="20"></meter><br>
<meter value="0.6">60%</meter>

    属性值：max（默认为1）、min（默认为0）、value（默认为0，可以是整数或浮点数）、optimum

    效果图：

    默认属性：

    meter{
    display:inline-block
}

  • progress标签（定义对象的下载进度）

    <progress value="20" max="100"></progress>

    属性值：max、min、value

    效果图：

默认属性：

progress{
    display:inline-block
}

  • time标签（定义时间和日期,常用来标记一篇文章的发布时间。注意：如果该time位于一个article中，那么它表示这篇文章的发布时间；如果不在article之中表示整个文档的发布时间。）

    <time datetime="2016-06-27">2016年6月27日</time><time>9:00</time>
<time datetime="2016-06-27T21：56：34TZD">2016年6月27日</time><time>9:00</time>

    其中尖括号内的datetime=”2016-06-27”是给机器看的，尖括号外的2016年6月27日是给用户看的

表单元素

  • tel(用于输入电话号码。)

  • search(用于提示用户输入要搜索的文字。search 和 text 之间的区别主要在于样式上)

  • url(用于输入单个网址。)

  • email(用于输入单个电子邮件地址,如果指定了 multiple 属性，就可以输入多个电子邮件地址，以逗号分隔)

  • datetime(用于输入时区设置为 UTC 的日期和时间。)

  • date(用于输入不含时区的日期。)

  • month(用于输入含年份和月份但不含时区的日期。)

  • week(用于输入含年份和周数但不含时区的日期。例如，此格式的日期 2011-W05 表示 2011 年的第 5 周。)

  • time(用于输入含小时、分钟、秒和秒的小数部分但不含时区的时间值。)

  • datetime-local(用于输入不含时区的日期和时间。)

  • number(用户输入数字)

  • range(用于输入数字，但与 number 的区别在于无需输入具体数字。在大部分支持该类型的浏览器中，范围控件的实施形式为滑块。)

  • color(用于通过颜色池控件选择颜色。)

  • label标签（搭配for使用，与后面定义的表单匹配，表现为表单前面的字）

  • datalist配合input使用，list属性配合datalist的id名，类似于下拉框

  • 表单属性：

autofocus

用于在网页加载后对焦到相关元素上的输入。

autofocus 的目标可以是输入、选择、文本区域和按钮。

placeholder

用于提示用户应输入的数据类型。

在对焦到相关元素以及用户输入数据之前，系统会以浅色文字显示占位符值。您可以在输入和文本区域中指定该值。

required

用于基本检测输入的类型是否正确，例如：对email的检测必须有“@”符号，对number类型不能输入字母

pattern

属于正则表达式,定义在input里面，对输入的元素进行验证

autocomplete

自动填充输入的内容，一般用于搜索框

音视频

audio(播放音频文件)、video(播放视频文件,相比于音频多个width属性)

autoplay

当打开网页时自动播放

controls

音频或视频的控制条

loop

循环播放设置

preload

提前加载

src

音视频路径，可相对也可绝对路径

source标签（配合在audio和video里面书写，表示不同的音视频路径）

embed标签（可以在里面插入插件）

<embed src="xxx.swf"></embed>

track( 标签为诸如 video 元素之类的媒介规定外部文本轨道。用于规定字幕文件或其他包含文本的文件，当媒介播放时，这些文件是可见的。)

<audio autoplay controls loop preload>
    <source src="audio1.mp3">
</audio>

<br><br><br><br><br><br>

<video autoplay controls loop preload width="500px">
    <source src="video1.avi" type="video/avi">
    <source src="video4.mp4" type="video/mp4">
    <!-- 播放带有字幕的视频 -->
    <track kind="subtitles" src="subs_chi.srt" srclang="zh" label="Chinese">
    <track kind="subtitles" src="subs_eng.srt" srclang="en" label="English">
</video>
