##mongodb
首先介绍一下homebrew：brew是一个Mac系统下的软件包管理工具，先安装brew，参考Homebrew官网的安装方法，终端下命令：
ruby -e “$(curl -fsSkL raw.github.com/mxcl/homebrew/go)”

安装完brew后先在终端更新一下：brew update

然后安装mongodb：brew install mongodb

###mongodb的使用（终端下的指令）

####在终端启动：

  • mongod —dbpath=数据库地址
  • 再在终端中键入mongo
    ####数据库的操作：

  • 1.创建数据库：use database_name

  • 2.查看所有数据库：show dbs

  • 3.查看当前使用的数据库：db 或 db.getName()

  • 4.删除数据库：db.dropDatabase()

  • 5.断开连接：exit

  • 6.查看命令api：help

####集合的操作

  • 1、查看帮助api：db.worker.help()
    ￼
  • 2、查看当前数据库下有哪集合（collection）：show collections
    ￼
  • 3、创建集合（collection）

//法1：db.createCollection(“collection_Name”)

//法2： db.collection_Name.insert(document)

注：两者的区别在于前者创建了一个空的集合，后者创建了一个空的集合并添加了一个Document数据

  • 4、删除当前数据库中的集合（collection）：db.collection_Name.drop()

  • 5、插入文档

使用insert()方法插入文档：

db.collection_name.insert(document)

db.worker.insert([{name:’wangWu’,age:50},{name:’xiaoMing’,age:60}])
￼
使用save()方法插入文档

db.collection_name.save(document)

注：如果不指定 _id 字段 save() 方法类似于 insert() 方法。如果指定 _id 字段，则会更新该 _id 的数据。

db.person.save({name:”xiaoHong”,age:50})

db.person.save({_id:ObjectId(“562c9caf671c978b6596e825”),name:”xiaoHong”,age:10})

  • 6、更新文档

update()方法用于更新已存在的文档

db.collection.update(query,update,{upset: boolean,multi: boolean,writeConcern: document})

参数说明：

query : update的查询条件，类似sql update查询内where后面的。

update : update的对象和一些更新的操作符（如$set,$inc…）等 $inc在原基础上累加后更新 $set直接更新

upsert : 可选，这个参数的意思是，如果不存在update的记录，是否插入objNew,true为插入，默认是false，不插入。

multi : 可选，mongodb 默认是false,只更新找到的第一条记录，如果这个参数为true,就把按条件查出来多条记录全部更新。

writeConcern :可选，抛出异常的级别。

实例

db.worker.update({name:’liSi’},{$set:{name:’liSi_update’}}) ￼
db.worker.update({name:’liSi_update’}, {$set: {age:40}},{multi:true}) ￼
db.worker.update({name:’liSi_update’},{$inc:{age:1}}) 2.save()方法通过传入的文档来替换已有文档

save（）方法实现文档更新：

db.collection.save(document,{writeConcern: document})

参数说明：
document : 文档数据。
writeConcern :可选，抛出异常的

实例

db.person.save({_id:ObjectId(“562c9caf671c978b6596e825”),name:”xiaoHong”,age:10})级别。
  • 7、文档删除

remove()方法是用来移除集合中的数据。

注：在执行remove()函数前先执行find()命令来判断执行的条件是否正确，这是一个比较。

语法：db.collection.remove（query，justOne)
如果你的 MongoDB 是 2.6 版本以后的

语法格如：db.collection.remove(query, {justOne: boolean,writeConcern: document}

参数说明：
query :（可选）删除的文档的条件。
justOne : （可选）如果设为 true 或 1，则只删除一个文档。
writeConcern :（可选）抛出异常实例

实例

db.worker.remove({name:’fJianZhou’}) 删除worker集合里name是fJianZhou的所有Document数据
db.person.remove({name:”xiaoHong”},1)  删除person集合里name是xiaoHong的第一条数据_name 集合的名字

  • 9、查询条件操作符
    ￼

（1）MongoDB (>) 大于操作符 - $gt

语法：db.collectionName.find({:{$gt:}})———-collectionName集合名词 key字段 value值

实例
db.worker.find({age:{$gt:30}})———– 查询age 大于 30的数据

￼
（2）MongoDB（>=）大于等于操作符 - $gte

语法:db.collectionName.find({:{$gte:}})———- collectionName集合名词 key字段 value值

实例:db.worker.find({age: {$gte: 30}})———– 查询age 3大于等于30 的数据

￼
（3）MongoDB (<) 小于操作符 - $lt

语法：db.collectionName.find( {:{$lt:}})———- collectionName集合名词 key字段 value值

实例：db.worker.find({age: {$lt: 30}})———— 查询age 小于30的数据

￼
（4）MongoDB (<=) 小于等于操作符 - $lte

语法：
db.collectionName.find({:{$lte:}})———– collectionName集合名词 key字段 value值

实例：
db.worker.find({age: {$lte: 30}}) ————查询age 小于等于30的数据

￼
（5）MongoDB 使用 (>=) 和 (<=) 查询 - $gte 和 $lte

语法：
db.collectionName.find({:{$gte:},:{$lte:}})———– collectionName集合名词 key字段 value值

实例：
db.worker.find({age: {$gte: 30, $lte: 50}}) ———–查询age 大于等于 30 并且 age 小于等于 50 的数据
￼

（6）MongoDB 等于（==）

语法：
db.collectionName.find({:,:}) ————- collectionName集合名词 key字段 value值

实例：
db.worker.find({“age”: 30})————查询age = 30的数据
￼

（7）MongoDB 使用 _id进行查询

语法：
db.collectionName.find({“_id” : ObjectId(“value”)}) ———–value _id的值

实例：
db.worker.find({“_id” : ObjectId(“562af23062d5a57609133974”)})———— 查询_id是 562af23062d5a57609133974 数据
￼

（8）MongoDB 查询某个结果集的数据的条数

语法：
db.collectionName.find().count() ————- collectionName集合名称

实例：
db.worker.find().count()
￼

（9）MongoDB 查询某个字段的值当中是否包含另一个值

语法：
db.collection.find({key:/value/}) ————– collectionName集合名称 key 字段 value值

实例：
db.worker.find({name:/value/}) ————-查询name里包含zhang的数据

￼（10）MongoDB 查询某个字段的值当中是否以另一个值开头

语法：
db.collection.find({key:/^value/})————collectionName集合名称 key 字段 value值

实例：
db.worker.find({name:/^zhang/})

（11）MongoDB 使用and和or条件查询
//and语法：db.worker.find({name:’tangcaiye’,age:30}) 查询name是tangcaiye并且age是30的数据

//or语法：db.worker.find({$or:[{age = 30},{age = 50}]}) 查询age = 30 或者 age = 50 的数据

//AND 和 OR 联合使用语法：

db.worker.find({name:’tangcaiye’,$or:[{age:30},{age:50}]})

  • 10、MongoDB Limit与Skip方法

1.Limit() 方法 ，读取指定数量的数据记录

语法：
db.collectionName.find().limit(number)——–collectionName集合 number读取的条数

实例：
db.worker.find().limit(3)—– 查询前3条数据
￼

2.MongoDB Skip() 方法 跳过指定数量的数据。

语法：
db.collectionName.find().skip(number)——–collectionName集合 number跳过的条数

实例：
db.worker.find().skip(3)——–查询3条以后的数据
￼

3.MongoDB Skip()方法和Limit()方法混合使用
注： 通常用这种方式来实现分页功能

语法
db.collectionName.find().limit(number).skip(number)

实例
db.worker.find().sort({age:-1})——- 查询在4-6之间的数据

  • 11、排序

MongoDB sort()方法可以通过参数指定排序的字段，其中 1 为升序排列，而-1是用于降序排列。

语法：

db.collectionName.find().sort({KEY:1}) 或者 db.collectionName.find().sort({KEY:-1})——-collectionName集合 key表示字段

实例：

db.worker.find().sort({age:1})——-查询出并升序排序 {age:1} age表示按那个字段排序 1表示升序

db.worker.find().sort({age:-1})——-查询出并降序排序 {age:-1} age表示按那个字段排序 -1表示降序
